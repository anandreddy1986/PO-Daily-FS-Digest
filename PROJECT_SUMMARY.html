🎯Use Case & Business Problem

The Challenge

As a Product Owner managing 4 File System teams (FS-Net, FS-local, FS-VFS, FS-GFS2) within the RHEL project, manually checking the Jira dashboard daily for updates across multiple teams was:

  • Time-consuming: 15-20 minutes every morning manually reviewing dashboards
  • Error-prone: Risk of missing critical CVEs or customer escalations
  • Inefficient: Repetitive manual queries for the same data patterns
  • Reactive: No proactive alerts for critical issues
  • Scattered: Information across multiple filters and views

The Solution

An intelligent, automated daily email digest that:

  • Consolidates key metrics from all 4 teams into a single email
  • Delivers actionable insights every morning at 7:00 AM IST
  • Highlights critical items: CVEs, customer escalations, testing status
  • Provides clickable JIRA links for immediate action
  • Uses smart grouping to reduce information overload

✨Key Features

1. Smart Scheduling & Reliability

  • Dual-trigger system: Cron (7:07 AM daily) + SessionStart hook (catch-up)
  • Mac-friendly: Automatically sends on first wake-up if scheduled time was missed
  • Deduplication: Never sends twice in one day using timestamp tracking
  • Durable cron: Survives Claude Code restarts

2. Intelligent Data Grouping

  • CVE consolidation: Groups same CVE across multiple RHEL versions (e.g., CVE-2026-31709 → 12 issues shown as 1 entry)
  • Prelim Testing grouping: Clusters related testing requests by description
  • Integration Testing grouping: Consolidates backport issues across versions
  • Space savings: Reduces email length by 60-70% while maintaining full information

3. Rich User Experience

  • Clickable links: Every JIRA issue is a hyperlink to the ticket
  • Hover tooltips: Mouse over version links to see status and assignee
  • Color-coded sections: Visual hierarchy for quick scanning
  • Professional signature: "Regards, Anand Reddy"
  • Clean subject line: "FS Teams Daily Digest - [Date]"

4. Comprehensive Coverage (7 Sections per Team)

  • 📌 New Issues: Created in last 24 hours
  • ✅ Closed Issues: Resolved in last 24 hours
  • 🔒 Active CVEs: Security vulnerabilities (grouped by CVE ID)
  • 🧪 Prelim Testing: Issues pending preliminary testing (grouped)
  • 🔬 Integration Testing: Issues in integration phase (grouped)
  • 🚨 Customer Escalations: Critical customer issues
  • 📈 Key Progress: High-priority items updated in last 24h
  • 📋 Tasks: Task-type issues (separate section)

🔍JQL Query Strategy

Section JQL Query Logic Purpose
New Issues created >= -24h AND type in (Bug, Vulnerability, Story) Track daily influx of new work
Closed Issues status changed to (Closed, Done, Resolved) DURING (-24h, now()) Measure daily throughput
Active CVEs type in (Vulnerability, Weakness) OR summary ~ "CVE" Security posture monitoring
Prelim Testing "Preliminary Testing" = Requested OR (status = "In Progress" AND labels = KWF:Task) Testing bottleneck visibility
Integration Testing status = Integration AND "Fixed in Build" is not EMPTY Release readiness tracking
Customer Escalations labels = "Escalation🔥" OR "Customer Impact" = "Customer Escalated" Customer satisfaction focus
Key Progress priority in (Blocker, Critical, Major) AND updated >= -24h Critical work momentum

🚀Implementation Highlights

Technical Achievements

  • Zero external dependencies: Uses corporate Red Hat SMTP, no third-party services
  • Regex-based grouping: Intelligent pattern matching for CVE and version extraction
  • Fault tolerance: Graceful degradation on API failures, partial reports on team errors
  • MCP integration: Leverages Model Context Protocol for secure credential management
  • Responsive HTML: Email renders correctly on desktop and mobile clients

Development Process

  • Built iteratively with user feedback loops
  • Started with Gmail SMTP, migrated to corporate Red Hat SMTP
  • Evolved from flat lists to intelligent grouping based on user feedback
  • Added catch-up mechanism after user reported Mac sleep scenarios
  • Refined JQL queries through multiple iterations to match dashboard filters

📚Lessons Learned

Key Insights

  • User feedback is gold: The grouped CVE format came from user observation that "same CVE appears multiple times"
  • Simple beats fancy: Red Hat SMTP (port 25, no auth) is more reliable than Gmail with app passwords
  • Context matters: Understanding the "CU+" flag mystery required seeing the actual dashboard, not just API data
  • Iterate quickly: Building in small increments with immediate testing caught issues early
  • Document as you go: JQL query evolution was easier to track with inline comments
